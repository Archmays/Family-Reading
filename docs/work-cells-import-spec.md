# 工作细胞导入规格说明

## 目标与类型

本规格用于将《工作细胞》加入 Book Companion / 家庭阅读助手，但不把项目改造成电子书阅读器、卷册目录站或学习进度产品。

新增内容类型为：

```json
{
  "contentType": "science-manga-companion"
}
```

`science-manga-companion` 表示“科学漫画伴读资料”：以家庭纸质书或已授权资料阅读为前提，围绕科学内容主题提供解释、问题、来源备注、图片证据和配套科学解释 prompt。

## 展示组织规则

- 《工作细胞》必须按“科学内容主题”展示，不按“卷”作为主导航。
- 卷和话只能作为来源备注，例如：`来源：第2卷 第8-9话`。
- 首页或系列入口应展示主题卡片，例如免疫、血液循环、病原体、健康教育等主题，而不是第1卷、第2卷入口。
- 主题页可以引用一个或多个来源话数。
- 前篇 / 后篇、上 / 下可以合并到同一个主题页，前提是它们讲的是同一个连续科学主题。
- `癌细胞` 和 `癌细胞Ⅱ` 是两个不同主题，不得错误合并；即使它们都与癌细胞知识相关，也必须保留独立主题记录和独立来源备注。

## 授权与公开发布边界

用户已确认购买并获得相关资料的使用与发布授权。在此授权范围内，允许公开发布：

- 漫画原图
- 从 EPUB 转换出的漫画页图片
- 缩略图
- 裁切图
- 主题伴读配套资料
- 科学解释 prompt 文本

不得因为版权保守假设而删除、隐藏或禁止使用已授权范围内的漫画页图片。

但完整 EPUB 原文件必须保持为私有输入素材，禁止进入任何公开发布位置，包括：

- `public/`
- `dist/`
- `build/`
- `docs/`
- GitHub Pages 发布目录
- 任何未来公开部署目录

## 私有输入与公开输出目录

当前已存在的 EPUB 输入素材位置是：

```text
source/工作细胞/
```

`source/` 已在 `.gitignore` 中忽略，应视为本地私有原始素材区。后续如新增 `source-private/` 或类似目录，也必须遵守同样规则：只作为私有输入，不直接发布。

公开派生资源应写入 `public/books/工作细胞/` 下，例如：

```text
public/books/工作细胞/<topic-slug>/pages/
public/books/工作细胞/<topic-slug>/thumbs/
public/books/工作细胞/<topic-slug>/crops/
public/books/工作细胞/<topic-slug>/topic.json
```

构建输出只允许复制公开派生资源，不允许复制完整 EPUB。任何复制规则、构建脚本或测试都必须显式防止 `.epub` 文件进入 `dist/`。

## 文案与术语规则

- 所有展示给家长和孩子看的内容必须是简体中文。
- 不允许机械繁简转换后直接发布。
- 生命科学、医学、健康教育术语必须核对大陆常用表达。
- 必须建立可人工修订的术语表，用于记录原始表达、建议简体中文表达、核对状态和备注。
- 术语表建议放在：

```text
docs/work-cells-terminology-review.md
```

术语表至少应包含：

- 原始词
- 简体中文建议词
- 科学分类
- 是否已人工核对
- 核对备注

## 音频规则

《工作细胞》当前定义为：

```json
{
  "hasAudio": false
}
```

工作细胞主题页不显示音频模块，不显示音频占位播放器，也不复用《不一样的卡梅拉》的音频逻辑。

如果未来另行确认有授权音频，应先修订本规格，再实现音频接入。

## 身体科学小站 Prompt 规则

【身体科学小站】只生成图片 prompt 文本，不生成图片，不调用图片生成工具。

每条 prompt 必须：

- 以 `【ChatGPT image】` 开头。
- 极其详细，足够生成一张独立原创科学解释图。
- 用于解释科学概念、身体机制、免疫过程、健康教育知识或漫画情节相关科学点。
- 使用原创视觉方案，不复刻《工作细胞》的官方角色、服装、构图、画风。
- 不要求生成与官方漫画相似的人物、制服、镜头、分镜或背景。

prompt 可以保存在主题数据中，也可以集中记录在：

```text
docs/work-cells-body-science-station-prompts.md
```

页面只展示 prompt 文本或将其作为配套资料保存；不得在本阶段生成图片文件。

## Draft Manifest 规则

在解析 EPUB 或提取图片之前，可以先根据用户提供的参考表建立 draft manifest。draft manifest 可以包含：

- 主题标题
- 主题 slug
- 科学分类
- 简体中文摘要草稿
- 来源备注，例如第几卷、第几话
- 前篇 / 后篇、上 / 下合并关系
- `癌细胞` 与 `癌细胞Ⅱ` 的独立主题记录
- `hasAudio: false`
- 待核对术语列表
- 身体科学小站 prompt 草稿文本

draft manifest 不得包含尚未生成或尚未确认存在的公开图片路径。

必须等 EPUB 文件实际存在并完成只读检查后才能完成：

- EPUB 内部目录确认
- 漫画页图片提取或转换
- 缩略图、裁切图生成
- 主题到具体页图的绑定
- 发布体积检查
- `.epub` 不进入公开目录的构建验证

## 不破坏卡梅拉规则

导入《工作细胞》不得破坏《不一样的卡梅拉》的现有行为：

- 卡梅拉继续按书籍入口展示。
- 卡梅拉书籍页的图片缩略图、放大图、问答证据图、背景图、百科图逻辑保持不变。
- 卡梅拉音频继续使用现有 `public/audio/carmela-s1/` 路径和 `preload="metadata"` 行为。
- 不向卡梅拉数据或页面引入阅读进度、打卡、统计、用户系统或排行榜。
- 不修改卡梅拉原始 source 资产。

## 验收标准

规格文档阶段验收：

- 已新增本规格文档。
- 未修改前端页面、构建脚本、测试脚本或公开数据。
- 未解析 EPUB，未提取图片，未生成图片。

后续实现阶段验收：

- 《工作细胞》公开数据使用 `contentType: "science-manga-companion"`。
- 《工作细胞》首页和系列入口按科学主题展示，不按卷展示。
- 卷和话只作为来源备注出现。
- `癌细胞` 与 `癌细胞Ⅱ` 是两个独立主题。
- 前篇 / 后篇、上 / 下按同一主题合并时保留完整来源备注。
- 《工作细胞》主题页设置 `hasAudio: false`，页面不显示音频模块。
- 所有家长和孩子可见内容均为简体中文。
- 科学术语已在可人工修订的术语表中核对。
- 身体科学小站只保存 prompt 文本，每条 prompt 均以 `【ChatGPT image】` 开头。
- prompt 用于原创科学解释图，不模仿《工作细胞》的官方角色、服装、构图、画风。
- 授权范围内的漫画页图片、缩略图、裁切图和配套资料可以发布。
- 完整 EPUB 原文件不在 `public/`、`dist/`、`build/`、`docs/` 或任何公开部署目录中。
- GitHub Pages 构建仍为静态站点，不引入服务器、数据库、登录或私有运行时服务。
- 《不一样的卡梅拉》现有页面、音频和图片逻辑通过回归检查。
