## GitHub Pages 发布判断

这些资源不适合原样直接进入 GitHub Pages 发布目录。

原因：

- 项目规则明确要求不要把原始 PDF 复制到公开发布目录。
- 单个原始 PDF 为 73.66 MiB，整套原始素材为 155.24 MiB；直接放入发布目录会明显增大仓库和 Pages 构建、下载负担。
- 12 个 MP3 单个大小约 4.52-8.70 MiB，单集音频可控，但整套音频一次性随页面发布会增加首轮分发压力。

建议：

- 原始 PDF 保留在 `source/`，后续如需页面使用，应先生成体积受控的派生素材，并写到 `source/` 之外。
- MP3 如需上线，优先使用压缩后的派生音频或外链对象存储/CDN；不要覆盖原始 MP3。
- 可按册或按内容分批发布音频，避免一次性把整套原始素材放入 GitHub Pages 目录。
- 不建议依赖 Git LFS 作为 GitHub Pages 常规静态资源方案，除非单独验证 Pages 构建和访问行为符合预期。
